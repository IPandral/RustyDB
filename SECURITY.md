# Security Policy

## Supported Versions

The following versions of RustyDB are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

> As RustyDB is currently in early development, only the latest version is supported.

## Reporting a Vulnerability

If you discover a security vulnerability in RustyDB, please follow the steps below:

1. **Do not open a public GitHub issue.** Public disclosure can put users at risk.
2. **Report it privately** by using [GitHub's private vulnerability reporting](https://github.com/IPandral/RustyDB/security/advisories/new).

### What to include in your report:
- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact of the vulnerability
- Any suggested fixes (optional but appreciated)

### What to expect:
- **Acknowledgement** of your report within **48 hours**
- **Status updates** every **7 days** while the issue is being investigated
- If the vulnerability is **accepted**, a fix will be worked on and you will be credited (if desired)
- If the vulnerability is **declined**, a clear explanation will be provided

Thank you for helping keep RustyDB secure! 🔒
